# Описание структуры базы данных

База данных backend-приложения `M'ART` построена на PostgreSQL и описана через Prisma Schema. Модель разделяет:

- технические сущности аутентификации;
- прикладной профиль пользователя;
- контент викторин;
- попытки прохождения;
- историю прогресса и наград;
- справочник достижений и факты их выдачи.

Ниже приведено актуальное описание таблиц и перечислений в соответствии с текущим файлом [schema.prisma](/c:/Users/Champ/Desktop/Projects/MART/elysia-app/prisma/schema.prisma:1).

## Таблица `user`

Таблица `user` хранит базовую учетную запись пользователя на уровне аутентификации.

| Столбец         | Тип        | Описание                                 |
| --------------- | ---------- | ---------------------------------------- |
| `id`            | `String`   | Первичный ключ пользователя.             |
| `name`          | `String`   | Имя пользователя на auth-уровне.         |
| `email`         | `String`   | Email пользователя, уникален.            |
| `emailVerified` | `Boolean`  | Признак подтверждения email.             |
| `image`         | `String?`  | Изображение пользователя на auth-уровне. |
| `createdAt`     | `DateTime` | Время создания записи.                   |
| `updatedAt`     | `DateTime` | Время последнего обновления записи.      |

## Таблица `profile`

Таблица `profile` описывает пользователя в прикладной части системы. В ней хранятся данные отображения и базовый агрегат `xp`.

| Столбец       | Тип        | Описание                                    |
| ------------- | ---------- | ------------------------------------------- |
| `id`          | `String`   | Первичный ключ профиля.                     |
| `userId`      | `String`   | Внешний ключ на `user.id`, уникален.        |
| `displayName` | `String?`  | Отображаемое имя пользователя в приложении. |
| `avatarUrl`   | `String?`  | Ссылка на аватар прикладного профиля.       |
| `xp`          | `Int`      | Накопленный опыт пользователя.              |
| `createdAt`   | `DateTime` | Время создания профиля.                     |
| `updatedAt`   | `DateTime` | Время последнего обновления профиля.        |

`level`, `totalQuizCompletions`, `totalQuizCorrectAnswers` и `lastRewardedAt` не хранятся в `profile` и вычисляются из доменных таблиц:

- уровень определяется из `xp`;
- число завершений берется из `quiz_attempt`;
- число правильных ответов берется из завершенных попыток;
- время последней награды берется из `progress_event`.

## Итоговая характеристика схемы

Текущая схема хранит:

- полноценную историю попыток прохождения викторин;
- пошаговое состояние активной попытки;
- историю конкретных ответов пользователя;
- базовый прогресс профиля через `xp`;
- журнал начисления XP;
- справочник достижений и факты их выдачи.

Такое разделение позволяет не смешивать игровой flow викторины, агрегаты прогресса и наградный пайплайн, а также упрощает дальнейшее расширение системы под новые источники XP.
